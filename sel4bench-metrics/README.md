# Collect condensed seL4 benchmark results

This action takes [sel4bench] test results and appends a condensed version to
the time series files in the [seL4/sel4bench-results] repository.

The action expects the test result json files generated by the
[sel4bench-hw][sel4bench-hw] action under `results/`, and the
[seL4/sel4bench-results] repository checked out under `sel4bench-results/` with
push rights.

[sel4bench]: https://github.com/seL4/sel4bench
[sel4bench-hw]: ../sel4bench-hw/README.md
[seL4/sel4bench-results]: https://github.com/seL4/sel4bench-results
[sel4bench-manifest]: https://github.com/seL4/sel4bench-manifest

## Content

The entry point is the script [steps.sh]. It reads the kernel and sel4bench
SHAs from the [sel4bench-manifest] at `manifest_sha`, runs the result
extraction, and commits and pushes the new data points to
[seL4/sel4bench-results].

[Build] and [platform] configurations are in the respective yaml and python
files in this repository. See these files for config documentation. They are
shared with the [sel4bench-hw] and sel4bench actions.

The main driver is [build.py --metrics].

[steps.sh]: ./steps.sh
[build.py --metrics]: ./build.py
[platform]: ../seL4-platforms/platforms.yml
[Build]: builds.yml

## Arguments

To change which metrics are extracted and where they are stored, edit
`metrics.yml` in [seL4/sel4bench-results].

- `manifest_sha`: the SHA of the manifest that was deployed after successful test
